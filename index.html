PaperSites

What

A small, opinionated Markdown → HTML generator with a custom block syntax and a clean, minimal theme. No JS framework, no build step beyond Python.

Why

Most static-site generators make you choose between plain markdown (no layout control) and a full templating language — Jinja, Liquid, partials, a front-matter schema with its own rules. PaperSites sits in between: a tiny block syntax for layout, regular markdown for everything else.

This homepage was built by running papersites build over this exact file.

Install

git clone https://github.com/<you>/papersites.git; cd papersites; pip install -e ".[dev]"

Requires Python 3.9+.

Build

papersites build     # reads ./content/*.md, writes ./site/

Serve

papersites serve      # serve ./site/ at http://localhost:8000

Writing pages

content/
  site.json     site title, logo, nav links
  index.md      -> site/index.html
  about.md      -> site/about.html
  assets/       copied to site/assets/ as-is

Each markdown file may start with front matter to set its own page title, the way this one does:

---
title: About me
---

# About
...

The suite includes regression tests for the bugs that turned up while hardening this from a single script into a package — a typography selector that never matched <ul>/<ol>, a navbar that pointed every link at the same href, and a parser that misread documentation about its own syntax as a real unclosed tag.
